SQLite优化方案

发表于 2015-07-18 | 分类于 OC

  1. 对查询进行优化，要尽量避免全表扫描，首先应考虑在 where 及 order by 涉及的列上建立索引。

  2. 应尽量避免在 where 子句中对字段进行null 值判断，否则将导致引擎放弃使用索引而进行全表扫描，如：select id from t where num is null。

    • 最好不要给数据库留NULL，尽可能的使用 NOT NULL填充数据库.
    • 备注、描述、评论之类的可以设置为 NULL，其他的，最好不要使用NULL。不要以为 NULL 不需要空间，比如：char(100) 型，在字段建立时，空间就固定了， 不管是否插入值（NULL也包含在内），都是占用 100个字符的空间的，如果是varchar这样的变长字段， null 不占用空间。
    • 可以在num上设置默认值0，确保表中num列没有null值，然后这样查询：select id from t where num=0

  3. 应尽量避免在 where 子句中使用 != 或 <> 操作符，否则将引擎放弃使用索引而进行全表扫描。

沙盒

发表于 2015-06-24 | 分类于 OC

沙盒机制

  • 每个iOS应用都有自己的沙盒，用来保存所有的非代码文件
  • 应用程序之间不能共享数据，沙盒里的文件不能被复制到其他应用程序文件夹中
  • 禁止读写沙盒以外的文件，禁止应用程序将内容写到沙盒以外的文件夹中
  • 应用程序请求的数据都要通过权限检测
  • 应用程序不能访问其他存储空间的内容

PrefixHeader设置

发表于 2015-05-09 | 分类于 Debug

运行新下载的项目或修改pch文件、info.plist文件位置后，Xcode可能会报一下错误：

  • ‘~PrefixHeader.pch’file not found（该文件不能被打开）
  • ‘~Info.plist’:The file “Info.plist” couldn’t be opened because there is no such file

这是文件的路径问题导致的，解决方法就是将绝对路径换成相对路径：

在Build Setting里查找Prefix Header，在其路径中用$(SRCROOT)替换掉工程名前的路径。（$(SRCROOT)表示当前目录下）。其实把项目名前的路径去掉也是可以的。
