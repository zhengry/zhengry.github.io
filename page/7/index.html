FMDB使用

2015-04-24
数据存储

FMDB以OC的方式封装了SQLite的C语言API。所以相对使用底层语言实现的Sqlite来说，FMDB主要有以下优点：

  • 使用起来更加面向对象，省去了很多麻烦、冗余的C语言代码；
  • 对比苹果自带的Core Data框架，更加轻量级和灵活；
  • 提供了多线程安全的数据库操作方法，有效地防止数据混乱；

FMDB有三个主要的类：

  1. FMDatabase

    一个FMDatabase对象就代表一个单独的SQLite数据库，用来执行SQL语句。

  2. FMResultSet

    使用FMDatabase执行查询后的结果集。

  3. FMDatabaseQueue

    用于在多线程中执行多个查询或更新，它是线程安全的。

block对对象的强弱引用

2015-03-29
block

要了解block对对象的强弱引用，首先要搞清楚block中可以引用的变量有哪些。

block中可以引用变量有以下几种：

  • 全局变量，包括静态变量
  • 被传入block的参数
  • 当前方法内的局部变量，会被捕获为const类型的变量。它们的值将会在声明block的地方被捕获。在嵌套block中，它们的变量值将会被最外层的block捕获。
  • 当前方法内的被__block标记的变量会被block引用，因此是变量是可变的。
  • 在block内部声明的变量，就是当做函数里的局部变量使用。

block的每次调用都会提供对变量的一份新的拷贝。这些变量早block范围内都会被当做const类型变量或被引用的变量。

block一开始是被放在栈上的，作为一个优化，Objective-C中会用Block_copy向block发送一个copy消息，block被copy到堆上。

block被拷贝到堆上时，会对block内部的对象变量产生强引用：

  • 如果你访问一个实例变量，block会对self产生一个强引用
  • 如果你访问一个实例变量的值，block会对这个变量产生一个强引用

UIImage相关问题

2014-08-20
渲染

  1. 直接引用图片文件夹到项目中，图片不展示的问题

    在外部创建好存放图片的imgs文件夹，然后拖放到工程中，选中copy选项，文件夹呈现为蓝色，此时在项目的Build Phases——>copy Bundle Resources 里，并没有对文件夹里的图片进行引用，此时引用的是文件夹，并没有对图片的直接引用。所以在使用文件夹里的图片时，UIImageView的图片不能展示出来。

  2. imageNamed:

On iOS 4 and later, if the file is in PNG format, it is not necessary to specify the .PNG filename extension. Prior to iOS 4, you must specify the filename extension.

imageNamed：后面的字符串默认的图片类型是png的，如果不是png的图片就需要带上图片类型后缀

当然还可以用：

官方：

If you have an image file that will only be displayed once and wish to ensure that it does not get added to the system’s cache, you should instead create your image using imageWithContentsOfFile:. This will keep your single-use image out of the system image cache, potentially improving the memory use characteristics of your app.

imageWithContentsOfFile:方法常用于只展示一次的图片，可以避免图片被加入到系统缓存中
