Xcode报错处理

发表于 2017-05-03 | 分类于 Debug

记录下最近自己在项目中遇到的Xcode报错及处理方法：

  1. 在podfile文件中删除了某个第三方库的引用，但cocoapods的xcconfig文件还保留着对该第三方库的配置，Xcode提示：

    file:///Users/…/XXXTests.debug.xcconfig: warning: Missing file: /Users/…/XXXTests.debug.xcconfig is missing from working copy

    解决方法：执行pod install方法后重新编译

  2. 在终端执行pod install或者pod update等命令后提示：

    [!] The `Project [Debug]` target overrides the `PODS_ROOT` build setting defined in `Pods/Target Support Files/Pods/Pods.debug.xcconfig’. This can lead to problems with the CocoaPods installation

    -Use the `$(inherited)` flag, or

    -Remove the build settings from the target.

    产生此警告的原因是项目 Target 中的一些设置，CocoaPods 也做了默认的设置，如果两个设置结果不一致，就会造成问题。

    解决方法：如果想要使用 CocoaPods 中的设置，分别在项目中定义PODS_ROOT 和 Other Linker Flags的地方，把他们的值用$(inherited)替换掉，进入终端，执行pod update，回到 Xcode，build通过。

  3. Duplicate symbol错误：

    产生这种错误是因为在工程总引入了重复的东西。图中标注的symbol后面就是表示重复的内容标志。

    解决方法：通过这个标志找到造成这个错误的文件或内容（通常是宏、枚举或类名），看自己是否重复引用该标志文件或重复定义了某个宏或枚举类型。

  4. Xcode提示：

    Undefined symbols for architecture i386:”OBJC_CLASS$_MBProgressHUD”, referenced from:`objc-class-ref in ViewController.o`

    ld: symbol(s) not found for architecture i386

    clang: error: linker command failed with exit code 1 (use -v to see invocation)

    出现的原因及解决方法：

    • 相关工程文件未导入

      在Build Phases的 Compile Source 中添加该工程文件

    • .framework文件未导入

      在Build Phases的 Link Binary With Libraries 中添加文件库

    • 文件路径缺失

      在使用CocoaPods导入第三方库时，如果文件路径设置不正确，通常会出现这个问题。在Building Setting的Library Search Paths中设置$(inherited)

    • 找不到对应的编译包

      在Build Phases的 Link Binary With Libraries 中添加 libc++ 库
