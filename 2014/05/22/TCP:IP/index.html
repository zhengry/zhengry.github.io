TCP/IP

2014-05-22
网络

计算机和网络设备要相互通信，双方就必须基于相同的方法。比如，如何探测到通信目标、由哪一边先发起通信、使用哪种语言进行通信、怎样结束通信等规则都需要事先确定。不同硬件、操作系统之间的通信，所有的这一切都需要一种规则——协议。TCP/IP是互联网相关的各类协议族的总称。

TCP/IP的分层管理

TCP/IP协议族按层次分别为：应用层、传输层、网络层和数据链路层。

  • 应用层：决定了向用户提供应用服务时通信的活动。

    比如：FTP（File Transfer Protocol，文件传输协议）、DNS(Domain Name System，域名系统)服务、HTTP协议

  • 传输层：提供处于网络连接中的两台计算机之前的数据传输。

    比如：TCP（Transmission Control Protocol，传输控制协议）、UDP（User Data Protocol，用户数据报协议）

  • 网络层：处理在网络上流动的数据包。该层规定了通过怎样的路径到达对方计算机，并把数据包传送给对方。 类似于快递员根据你的地址（MAC地址）选择合适的路线（传输路线）把包裹（数据包）送到你家。

  • 数据链路层：用来处理连接网络的硬件部分。

    比如网卡、光纤、硬件的设备驱动等。

分层的好处：高内聚，低耦合。如果只有一个协议，某个地方需要修改设计时，需要把所有部分整体替换掉。但是分层之后，只需要修改要变动的层。另外，各层也只需要考虑分派给自己的任务。

TCP/IP通信传输流

利用TCP/IP协议族进行网络通信时，会通过分层顺序与对方进行通信。发送端从应用层往下走，接收端则从数据链路层往上走。

以HTTP为例，发送端在层与层之间传输数据时，每经过一层必定会被打伤一个该层所属的首部信息（header）。反之，接收层在层与层之间呢传输数据时，每经过一层时会把对应的首部消去。

应用层的HTTP数据在经过传输层（TCP Layer）的时候，会加上TCP的header，变成了TCP Segment；传输层的Segment在经过网络层（IP Layer）的时候，会加上IP的header，变成一个IP Packet；网络层的IP Packet在经过链路层（Link Layer）的时候会加上Link Layer的header，变成一个Frame，最后Frame会在物理层将数字信号转化为物理信号传输。（Segment、Packet、Frame都是不同层的包，英译汉之后它们都叫包。。。）

所以对每层的学习，就是要学习每一层的header。
